# 모바일 상세 페이지 프리뷰 폭 개선 설계서

## 배경
모바일(예: 375px)에서 리스트 카드 프리뷰보다 상세 페이지 프리뷰가 더 작게 보입니다. 디바이스 폭 한계로 절대 크기를 늘릴 수는 없지만, 최소한 리스트 뷰와 동일한 폭으로 감상할 수 있도록 레이아웃을 조정합니다.

## 목표
- 모바일에서 상세 페이지 프리뷰 폭을 리스트 카드와 동일하게 맞춘다.
- 데스크톱/태블릿 레이아웃은 기존과 동일하게 유지한다.
- 가로 스크롤 없이 안정적으로 렌더링한다.

## 범위
- 대상 파일: `src/components/PostDetail.tsx`
- 비대상: 렌더러 로직, 이미지/캔버스 자체 스케일 로직

## 현상 요약
리스트 카드(모바일 375px):
- 메인 패딩 `px-4` → 카드 폭 약 343px
- 프리뷰 내부 `p-4` → 렌더 영역 약 311px

상세 페이지(모바일 375px):
- 메인 패딩 `px-4` → 343px
- `article p-6` → 내부 폭 295px
- 프리뷰 내부 `p-4` → 렌더 영역 263px

결과적으로 상세 페이지 렌더 영역이 리스트 대비 약 48px 더 작음.

## 원인
상세 페이지는 `main px-4` + `article p-6` + 프리뷰 내부 `p-4`의 중첩 패딩으로 인해 렌더 영역이 과도하게 축소됨.

## 해결 방향(권장)
모바일에서 프리뷰만 풀폭처럼 보이도록 확장하고, 텍스트 영역은 기존 패딩을 유지한다.

### 핵심 아이디어
- `article` 패딩을 모바일에서 `p-4`, 데스크톱에서 `p-6`로 반응형 처리
- 프리뷰 래퍼에 모바일 `-mx-4`를 적용해 `main`의 가로 패딩만큼 확장
- 데스크톱에서는 `mx-0`으로 복원

## 설계안(권장안)
1. `article` 패딩을 `p-4 sm:p-6`으로 조정
2. 프리뷰 래퍼에 `-mx-4 sm:mx-0` 추가
3. 프리뷰 내부 `p-4`는 유지하여 리스트와 렌더 영역 폭을 동일화

## 대안
1. 프리뷰 내부 `p-4`를 모바일에서 축소 (`p-2`)  
   - 장점: 간단
   - 단점: 리스트와 내부 여백이 달라져 시각적 일관성 저하
2. 상세 페이지 전체를 풀폭 레이아웃으로 전환  
   - 장점: 최대한 넓은 프리뷰 확보
   - 단점: 텍스트 가독성 저하, 데스크톱 레이아웃 파손 가능

## 구현 단계
1. `PostDetail`의 `article` 패딩 반응형 적용
2. 프리뷰 래퍼에 모바일 음수 마진 적용
3. 데스크톱/태블릿 레이아웃 확인

## 수용 기준(AC)
- 375px에서 상세 프리뷰 폭이 리스트 프리뷰 폭과 동일해야 한다.
- 768px 이상에서 레이아웃 변화가 없어야 한다.
- 가로 스크롤이 발생하지 않아야 한다.

## QA 체크
1. 모바일(375px) 리스트 카드 프리뷰 폭 확인
2. 동일 포스트 상세 페이지에서 프리뷰 폭 비교
3. 768px/1024px에서 레이아웃 유지 확인
4. 스크롤 및 오버플로우 유무 확인
