# 상세 페이지 데이터 바인딩 작업 지시서

## 목표
상세 페이지에서 실제 게시물 데이터를 API로 가져와 렌더링하고, 댓글은 기능 개발 전까지 “댓글이 없습니다” 상태로 표시한다.

## 범위
- 상세 페이지 데이터 패칭 및 바인딩
- 댓글 섹션 기본 상태 UI(댓글 없음)

## 비범위
- 댓글 작성/삭제/투표 기능
- 서버 API 확장 또는 스키마 변경
- SEO/메타데이터 추가

## 적용 위치
- 라우트: `src/app/posts/[id]/page.tsx`
- API: `src/app/api/posts/[id]/route.ts`
- 타입: `src/types/post.ts`
- 필요 시 컴포넌트 분리: `src/components/PostDetail.tsx`

## 작업 내용

### 1) 상세 페이지 실제 데이터 바인딩
- `posts/[id]` 라우트에서 **실제 데이터**를 로드한다.
- 서버 컴포넌트 기준 `fetch('/api/posts/[id]')` 또는 직접 DB 접근 패턴 중 프로젝트 기준에 맞춰 사용.
- API 응답 필드를 상세 페이지 UI에 매핑.
- 데이터가 없을 경우(404/에러) 사용자에게 안내 문구 표시.

### 2) 댓글 섹션 “댓글이 없습니다” 상태
- 댓글 기능은 미개발 상태이므로, 댓글 섹션에 기본 상태를 표시한다.
- 표시 문구 예시: `댓글이 없습니다` 또는 `No comments yet` (프로젝트 언어 톤에 맞춤)
- 댓글 입력 영역은 숨기거나 비활성 안내 문구로 대체.

## 구현 단계
1. 상세 페이지에서 `id` 파라미터를 사용해 데이터 요청 경로 구성.
2. API 응답을 UI에 바인딩하고, 로딩/에러 상태 처리.
3. 댓글 섹션에 기본 “댓글 없음” 상태를 적용.
4. 모바일/데스크탑 모두에서 레이아웃 이상 없는지 확인.

## 완료 기준
- 상세 페이지가 실제 게시물 데이터를 렌더링한다.
- 데이터가 없을 때 적절한 안내가 표시된다.
- 댓글 섹션은 “댓글이 없습니다” 상태로 표시된다.
